## 1. Executive Summary

AskAdil aims to build a curated, searchable directory of Muslim/Islamic-specialist solicitors so the chatbot can refer users to appropriate legal help. This document captures comprehensive research into the current landscape, available data sources, and a phased roadmap to build the directory.

**Key finding:** No public, searchable Muslim lawyer directory exists in the UK. AskAdil would be the first of its kind.

## 4. Data Sources for Expansion

### 4.1 Official Registers

#### SRA Solicitors Register (Primary Source)

- **URL:** https://www.sra.org.uk/consumers/register/
- **API:** https://sra-prod-apim.developer.azure-api.net/
- **Format:** JSON, updated every 24 hours
- **Content:** All ~200,000+ solicitors in England & Wales — name, firm, location, practice areas, SRA number, regulatory status
- **Limitation:** No religion field. Must infer via name matching or cross-referencing.
- **Access:** Public data, free API. Registration required for API key.

#### Bar Standards Board Barristers' Register

- **URL:** https://www.barstandardsboard.org.uk/for-the-public/search-a-barristers-record/the-barristers-register.html
- **Format:** Web-based search only, no public API
- **Content:** All practising barristers — name, chambers, practising status, disciplinary findings
- **Limitation:** No religion field. Web scraping would be needed for bulk extraction.

#### Bar Council - Find a Barrister

- **URL:** https://www.barcouncil.org.uk/find-a-barrister.html
- **Format:** Web-based search
- **Useful for:** Finding barristers by specialism and location

### 4.2 Name-Matching Approach

**Concept:** Query the SRA register with common Muslim surnames to identify potential Muslim solicitors for outreach.

**Common Muslim surnames for matching:**

| High frequency | Medium frequency | Lower frequency |
|---------------|-----------------|-----------------|
| Khan, Ahmed/Ahmad, Ali, Hussain/Hussein, Malik, Sheikh/Shaikh, Syed/Saeed | Rahman/Rehman, Begum, Chowdhury, Iqbal, Hasan/Hassan, Qureshi, Ansari, Mirza | Bukhari, Farooq, Siddiqui, Usman, Rashid, Nawaz, Patel*, Baig, Dar |

*Patel spans Muslim, Hindu, and Jain communities — high false positive rate.

**Limitations:**
- **False positives:** Names like Ali, Khan, Patel span multiple faiths and cultures
- **False negatives:** Muslim converts (e.g. Jonathan Butterworth chairs IAMLA), diverse ethnic backgrounds, anglicised names
- **Ethical:** Profiling by name is sensitive. Must be used for *outreach invitations*, not auto-listing
- **Legal:** GDPR applies. SRA data is public but inferring religion from names requires careful handling
- **Accuracy estimate:** ~70-80% precision for top-frequency surnames, much lower for ambiguous ones

**Recommended use:** Lead generation for outreach only. Never auto-list. Always require opt-in consent.

**Referral matching logic:**
```
User says: "I need help with Islamic divorce in Manchester"
→ Match: specialism=islamic_divorce, location=Manchester
→ Surface: Aramas Family Law (specialist, Chambers-ranked, Manchester)
→ Also: O'Donnell Solicitors (Oldham), MWG Solicitors (Manchester)
```

### Phase 2: Outreach & Consent (April — June 2026)

**Goal:** Contact priority firms, verify details, obtain consent to be listed.

| Task | Detail |
|------|--------|
| Outreach email template | Draft professional email from MCB/AskAdil explaining the directory, asking for consent |
| Priority outreach list | Start with Tier A firms (I Will Solicitors, Aramas, White Horse, Kuddus, Ascentim, Batley Law, Rahman Lowe, Blakewater) |
| Consent tracking | Update `consent_to_list` and `outreach_status` fields in the JSON for each firm contacted |
| Verification checklist | For each firm: confirm team members, practice areas, languages, contact preferences, free consultation availability, legal aid/no-win-no-fee |
| Network outreach | Contact AML, MLAG, Muslim Lawyers' Hub — introduce AskAdil, ask them to promote to members |

**Outreach data to collect per firm:**
- Consent to be listed on AskAdil (yes/no/conditions)
- Preferred contact method for referrals (phone/email/form)
- Free initial consultation? (yes/no/duration)
- Legal aid available? (yes/no/which areas)
- No-win-no-fee available? (yes/no/which areas)
- Languages spoken (verified)
- Current team members (verified)
- Geographic coverage (verified)
- Any specialisms to add or remove
- Would they like to receive referral notifications?

### Phase 3: Self-Registration & SRA Integration (June — September 2026)

**Goal:** Scale the directory beyond manual curation.

| Task | Detail |
|------|--------|
| "List your practice" page | Self-registration form on AskAdil website for lawyers to opt in |
| Promote via networks | AML, MLAG, Muslim Lawyers' Hub, MCB affiliates, mosque networks |
| SRA API integration | Register for API key. Build pipeline to query solicitors by name/location/practice area |
| Name-matching prototype | Query SRA with top Muslim surnames. Cross-reference with practice areas relevant to AskAdil |
| Outreach from SRA matches | Send invitations to identified solicitors: "We found you on the SRA register. Would you like to be listed on AskAdil?" |
| Admin panel | Simple CRUD interface for managing directory entries (add/update/remove firms, track consent, log outreach) |

**SRA API integration spec:**
```
1. Register at https://sra-prod-apim.developer.azure-api.net/
2. Query: GET /organisations?city=Manchester&practiceArea=family
3. Query: GET /individuals?surname=Khan&city=Birmingham
4. Store results in Postgres (not JSON file)
5. Match against Muslim surname list
6. Generate outreach candidate list
7. Never auto-list — always require opt-in
```

## 7. Legal & Ethical Considerations

### GDPR & Data Protection
- All firm contact details used are publicly available on firm websites
- SRA register data is public domain
- Name-matching to infer religion requires careful handling — used for outreach invitations only, never auto-listing
- Consent must be explicit and recorded before listing any firm
- Right to removal must be honoured promptly

### Disclaimers Required
- "Listing does not constitute endorsement by MCB or AskAdil"
- "Verify a solicitor's status at sra.org.uk before instructing"
- "AskAdil provides information only and is not a regulated legal referral service"

### Regulatory
- AskAdil is NOT a regulated referral service and must not present itself as one
- No referral fees may be charged to or received from listed firms (SRA rules)
- Must comply with Legal Services Act 2007 regarding legal services marketing

### Name-Matching Ethics
- Must never auto-list based on name inference
- Name matching is a lead generation tool for invitations only
- Must acknowledge limitations (false positives, false negatives, converts, diverse backgrounds)
- All communications must frame it as an invitation, not an assumption about identity
